# Интеллектуальное зонированное освещение двора на базе Frigate NVR и Home Assistant

Репозиторий содержит конфигурацию автоматизации для умного дома (Home Assistant), реализующую динамическое управление уличным прожектором на основе объектной видеоаналитики от Frigate NVR.

## 📋 Описание проекта

В отличие от классического включения уличного освещения по таймеру или датчикам движения, данное решение использует компьютерное зрение для детекции конкретных типов объектов (*в данном случае — человека*) в строго заданной геозоне двора. Это позволяет экономить электроэнергию, снизить световое загрязнение и автоматизировать освещение только там, где это действительно необходимо в данный момент.

### Ключевые особенности
* **Объектная аналитика:** Исключены ложные срабатывания от качающихся деревьев, домашних животных, птиц или пролетающих насекомых.
* **Устойчивость к перезагрузкам:** Архитектура построена на `Trigger ID`, что предотвращает «зависание» включенного света при перезапуске сервера Home Assistant.
* **Учет присутствия:** Система активна только тогда, когда кто-то из членов семьи находится дома.
* **Умный ручной режим (Override):** Специальный виртуальный тумблер позволяет принудительно включить свет (например, для разгрузки автомобиля), блокируя автоматику, но корректно возвращая управление системе при его выключении.

## ⚙️ Архитектура и режимы работы

Система поддерживает три основных режима работы, логика которых разделена между двумя независимыми автоматизациями:


              ┌───────────────────────────┐
              │   Движение в зоне yard    │
              └─────────────┬─────────────┘
                            │
             [Кто-то есть дома? (Группа)]
                            │ Да
           [Тумблер ручного режима выкл?]
                            │ Да
             [За 30 мин до заката / Ночь]
                            │ Да
                            ▼
              ┌───────────────────────────┐
              │ Включение Projector_stolb_2│
              └───────────────────────────┘



1. **Автоматический режим (Детекция):** При обнаружении человека в зоне `yard` камеры `road_gate` в темное время суток (начиная за 30 минут до заката и до рассвета) включается прожектор `switch.projector_stolb_2`. Свет автоматически гаснет через 2 минуты и 30 секунд после того, как зона становится пустой.
2. **Режим «Никого нет дома»:** Если все трекеры семьи находятся вне дома, автоматика полностью блокируется. Это исключает работу освещения от редких ложных срабатываний нейросети в отсутствие хозяев.
3. **Режим «Погрузка / Постоянный свет»:** При активации вспомогательного тумблера `input_boolean.yard_light_manual_mode` свет включается принудительно на постоянную основу. При отключении тумблера система проверяет: если во дворе пусто — свет гаснет сразу, если там остался человек — свет продолжит гореть и погаснет только через 2.5 минуты после его ухода.

## 🚀 Пошаговое развертывание

### Шаг 1: Настройка зоны во Frigate NVR
В файле конфигурации Frigate (`frigate.yml`) для вашей камеры необходимо разметить зону `yard` и указать отслеживание объекта `person`.

```yaml
cameras:
  road_gate:
    ffmpeg:
      inputs:
        - path: rtsp://192.168.1.X:554/stream
          roles:
            - detect
    zones:
      yard:
        coordinates: 0.11,0.45,0.85,0.43,0.92,0.88,0.05,0.91 # Замените на ваши координаты
    objects:
      track:
        - person

```

### Шаг 2: Создание группы отслеживания в Home Assistant

Для работы логики присутствия создайте группу, объединяющую устройства членов семьи. Добавьте в ваш `configuration.yaml`:

```yaml
group:
  all_family_trackers:
    name: "Все трекеры семьи"
    entities:
      - device_tracker.user1_phone
      - device_tracker.user2_phone

```

### Шаг 3: Создание виртуального тумблера (Helper)

Для ручного управления необходимо создать виртуальный переключатель через веб-интерфейс Home Assistant:

1. Перейдите в **Настройки** -> **Устройства и службы** -> **Вспомогательные элементы**.
2. Нажмите **Создать вспомогательный элемент** -> **Переключатель (Toggle)**.
3. Укажите имя: `yard_light_manual_mode` (разрешите отображение как `input_boolean.yard_light_manual_mode`).

### Шаг 4: Интеграция автоматизаций

Скопируйте представленный ниже код автоматизаций в ваш файл `automations.yaml` или импортируйте через встроенный YAML-редактор в интерфейсе Home Assistant.

## 💻 Код автоматизаций

### 1. Основная автоматизация управления по детекции

```yaml
id: yard_lighting_person_detection_v3
alias: "Освещение двора по детекции человека"
description: "Управление прожектором через Trigger ID с проверкой присутствия и ручного обхода."
mode: single # Используем single, так как логика разделена на атомарные триггеры, состояние не должно сбрасываться

trigger:
  # Сигнал от Frigate: человек вошел в зону
  - platform: state
    entity_id: binary_sensor.yard_person_occupancy
    to: "on"
    id: "motion_start"

  # Сигнал от Frigate: зона пуста непрерывно в течение 2.5 минут
  - platform: state
    entity_id: binary_sensor.yard_person_occupancy
    to: "off"
    for:
      minutes: 2
      seconds: 30
    id: "motion_end"

action:
  - choose:
      # =========================================================================
      # ВЕТКА ВКЛЮЧЕНИЯ СВЕТА
      # =========================================================================
      - alias: "Включение от детектора"
        conditions:
          # Проверяем, что сработал триггер начала движения
          - condition: trigger
            id: "motion_start"
          
          # Автоматика работает только если ручной режим ПОСТОЯННОГО света выключен
          - condition: state
            entity_id: input_boolean.yard_light_manual_mode
            state: "off"
          
          # Шаблон проверки присутствия: хотя бы один объект из группы должен быть дома
          - condition: template
            value_template: |
              {% set home_entities = expand('group.all_family_trackers') 
                | selectattr('state', 'eq', 'home') 
                | list %}
              {{ home_entities | length > 0 }}
          
          # Условие освещенности: период времени между "за 30 минут до заката" и "рассветом"
          # Использование оператора OR решает проблему сброса условия после полуночи (00:00)
          - condition: or
            conditions:
              - condition: sun
                after: sunset
                after_offset: "-00:30:00"
              - condition: sun
                before: sunrise
        sequence:
          - action: switch.turn_on
            target:
              entity_id: switch.projector_stolb_2

      # =========================================================================
      # ВЕТКА ВЫКЛЮЧЕНИЯ СВЕТА
      # =========================================================================
      - alias: "Выключение от детектора"
        conditions:
          # Проверяем, что сработал триггер отсутствия движения (таймаут истек)
          - condition: trigger
            id: "motion_end"
          
          # Если включен ручной режим, автоматика не имеет права гасить свет
          - condition: state
            entity_id: input_boolean.yard_light_manual_mode
            state: "off"
        sequence:
          - action: switch.turn_off
            target:
              entity_id: switch.projector_stolb_2

```

### 2. Автоматизация сервисного тумблера ручного обхода

```yaml
id: yard_lighting_manual_switch
alias: "Освещение двора: Принудительный тумблер"
description: "Обеспечивает работу постоянного освещения в обход автоматики детекции."
mode: restart # Позволяет мгновенно переключать логику при частых ручных щелчках тумблера

trigger:
  # Отслеживаем ручное включение виртуального тумблера на дашборде
  - platform: state
    entity_id: input_boolean.yard_light_manual_mode
    to: "on"
    id: "switch_on"

  # Отслеживаем ручное выключение виртуального тумблера на дашборде
  - platform: state
    entity_id: input_boolean.yard_light_manual_mode
    to: "off"
    id: "switch_off"

action:
  - choose:
      # =========================================================================
      # ТУМБЛЕР ВКЛЮЧЕН: Активация постоянного света
      # =========================================================================
      - alias: "Включение тумблера"
        conditions:
          - condition: trigger
            id: "switch_on"
        sequence:
          - action: switch.turn_on
            target:
              entity_id: switch.projector_stolb_2

      # =========================================================================
      # ТУМБЛЕР ВЫКЛЮЧЕН: Возврат в автоматический режим
      # =========================================================================
      - alias: "Выключение тумблера"
        conditions:
          - condition: trigger
            id: "switch_off"
          
          # Защита от выключения света "на человека":
          # Прожектор погаснет сразу только если Frigate подтверждает, что во дворе НИКОГО НЕТ.
          # Если человек в кадре есть, эта ветка пропустит выполнение, и свет погасит 
          # основная автоматика через 2.5 минуты после того, как человек уйдет.
          - condition: state
            entity_id: binary_sensor.yard_person_occupancy
            state: "off"
        sequence:
          - action: switch.turn_off
            target:
              entity_id: switch.projector_stolb_2

```
## 🛠️ Рекомендации по оптимизации детекции (Борьба с ложными срабатываниями)

Для минимизации ложных включений прожектора в ночное время, настройте следующие параметры в `frigate.yml`:

1. **Маски движения (`motion -> mask`):** Закройте масками кроны деревьев и кусты, качающиеся на ветру в поле зрения камеры. Детектор объектов обрабатывает только те зоны, где зафиксировано изменение пикселей (движение).
2. **Фильтры размеров (`objects -> filters`):** Укажите минимальные (`min_area`) и максимальные (`max_area`) размеры для объекта `person`. Это предотвратит срабатывания от пауков, ползающих по объективу камеры ночами, или от птиц.
3. **Порог уверенности (`threshold`):** Для ночного времени рекомендуется поднять порог детекции `threshold` до `0.75 - 0.80`, чтобы снизить влияние цифрового шума матрицы при слабом освещении.
