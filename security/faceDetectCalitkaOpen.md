# 🚪 Frigate - Автоматическое открытие калитки для известных персон

Автоматизация открывает калитку и отправляет уведомление в Telegram при обнаружении известных лиц с точностью >90%.

## 📋 Описание

При обнаружении знакомого лица на камере `doorbell` система:
1. Проверяет, что калитка закрыта (защита от дребезга)
2. Открывает калитку (переключает реле)
3. Отправляет фото с камеры в Telegram
4. Блокирует повторные срабатывания на 2 минуты

## 🎯 Триггеры

| Тип | Источник | Условие |
|-----|----------|---------|
| MQTT | `frigate/tracked_object_update` | Событие распознавания лица |

## 🔧 Зависимости

| Компонент | Назначение | Где находится |
|-----------|------------|---------------|
| Frigate | Распознавание лиц | Отдельный сервер (10.98.98.5) |
| `binary_sensor.opensensor_kalitka_contact` | Датчик открытия калитки | Конфигурация Zigbee |
| `switch.kalitka_door_relay_0` | Реле управления калиткой | Конфигурация |
| Telegram bot | Отправка уведомлений | Интеграция HA |

## 📁 Файл автоматизации

- [`faceDetectCalitkaOpen.yaml`](faceDetectCalitkaOpen.yaml) — основная автоматизация


## ⚙️ Логика работы

### Условия срабатывания (ВСЕ должны быть истинны):

1. **Калитка закрыта** минимум 20 секунд (защита от дребезга)
2. **Камера** — `doorbell` (только калитка)
3. **Тип объекта** — `face` (лицо)
4. **Имя** в списке разрешенных:
   ```
   ['Dmitry', 'Katy', 'Maxim', 'Pancho', 'Teshcha', 'Sanya', 'Kiseleva']
   ```
5. **Точность распознавания** ≥ 90% (score ≥ 0.9)
6. **Наличие снимка** во Frigate (`has_snapshot: true`)
7. **Защита от частых срабатываний** — не чаще раза в 2 минуты

### Действия:

1. **Открытие калитки** — переключение реле `switch.kalitka_door_relay_0`
2. **Отправка фото** в Telegram (параллельно с открытием):
   - Фото берется напрямую из Frigate API
   - Подпись содержит имя, точность и время
3. **Задержка 2 секунды** (стабилизация)

## 🎛 Настройка

### Список разрешенных лиц
Отредактируйте массив в условии:
```yaml
{% set allowed_persons = ['Дмитрий', 'Катя', 'Максим', 'Новый_человек'] %}
```

### Изменение таймаутов
- **Защита от дребезга** — параметр `seconds: 20` в первом условии
- **Частота срабатываний** — параметр `> 120` (2 минуты)

### Камера
Если нужно другое название камеры, измените:
```yaml
message.camera == 'doorbell'
```

## 🔍 Отладка

### Проверка MQTT-событий
Подпишитесь на топик в MQTT Explorer:
```
frigate/tracked_object_update
```

### Принудительный сброс защиты
Если нужно сбросить таймер последнего срабатывания:
```bash
# В терминале HA или через Developer Tools
ha core restart
```
Или в **Developer Tools → Actions** выполнить:
```yaml
action: automation.reload
target:
  entity_id: automation.frigate_avtomaticheskoe_otkrytie_kalitki_dlia_izvestnykh_person
```

## 🐛 Возможные проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Калитка не открывается | Датчик показывает 'on' | Проверьте датчик открытия |
| Нет фото в Telegram | `has_snapshot: false` | Проверьте настройки Frigate |
| Лицо не распознается | Низкий score или имя не в списке | Проверьте точность и список |

## 📌 Примечания

- Используется **`chat_id`** вместо устаревшего `target` (совместимо с HA 2026.9+)
- Фото берется с параметром `crop=1` для обрезки по лицу
- Качество снимка: `quality=95`
- Автоматизация работает в режиме `single` (не накапливает очередь)
